# Support

Thank you for using `docker-flask-postgres-api`.

## Getting help

If you run into a problem, start with the following:

- Check the README for setup and usage instructions.
- Review the issue tracker for known bugs and existing discussions.
- Inspect container logs with `docker compose logs`.
- Confirm your environment variables and database settings.

## What to include in a support request

Please include:

- Your operating system.
- Docker and Docker Compose versions.
- Python version, if applicable.
- The exact command you ran.
- The full error message or logs.
- Steps to reproduce the issue.

## Security issues

Do not post secrets, private keys, or production credentials in public issues. For security-related problems, use a private reporting channel if one is available.

## Scope

Support is best-effort and focused on:

- Local development.
- Containerized deployment.
- Flask application behavior.
- Postgres connectivity and migrations.
