# Contributing

Thank you for contributing to `docker-flask-postgres-api`.

## Ways to contribute

- Report bugs.
- Suggest improvements.
- Submit pull requests.
- Improve documentation.
- Add tests or examples.

## Before you start

Please:

- Read the README and existing issues.
- Check whether the change already exists.
- Keep changes focused and small when possible.
- Follow the project code style.

## Development setup

1. Fork the repository.
2. Clone your fork.
3. Create a virtual environment if needed.
4. Start the app with Docker Compose.
5. Make your changes.
6. Run tests and lint checks.
7. Submit a pull request.

## Pull request guidelines

- Use a clear title and description.
- Explain what changed and why.
- Link related issues when relevant.
- Include screenshots or logs if they help explain the change.
- Keep PRs limited to one purpose when possible.

## Code quality

- Write readable Python code.
- Add or update tests for behavior changes.
- Update documentation when user-facing behavior changes.
- Avoid committing secrets or generated artifacts.

## Commit messages

Use short, meaningful commit messages, for example:

- `fix: handle missing database connection`
- `docs: improve setup instructions`
- `test: add API health check coverage`
